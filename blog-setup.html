Setting up a blog on Github

Published: Fri 26 June 2020
By Kishan Sampat

In Learning Path.

Blogging can be a great way to take notes on your learnings. It’s like taking notes in a class when you were in college. I find it easier to remember things when I write them down. I was suggested to use Pelican as it is quite simple to deploy, and is also written in Python. This will further help me accelerate my Python learning. I will describe below on how to set up Pelican on your environment and start blogging away.

I found Pelican’s setup documentation to be somewhat inadequate at explaining the full setup, so I also used a couple of sites to get the blog going. I used a combination of
these sites: Renata’s Blog, Dataquest & Just Alfred. While all of them have different approaches to how your setup should be, the initial setup is common amongst them, and I used what was best suited for me. It is not necessary that you follow the setup as below, and you can feel free to use your own combination or suggest a better one (I’m always happy to receive feedback!)

Setting up your Github Repo

To start off, it is good practice to host your blog on Github. It’s free, easy and quick to set up. You can follow the instructions on the Github Help Page to set it up. For your blog name, I’d recommend starting with your own name, for example, I set up mine as kishansampat.github.io

And as mentioned in [Renata’s Blog]:(https://rsip22.github.io/blog/create-a-blog-with-pelican-and-github-pages.html) & the Github Help Page, it is recommended to initialize the repository with a README and a free software license, so that you still own the code, but you make sure that others will also benefit from it, by allowing them to study it, reuse it and share it.
